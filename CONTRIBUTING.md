# Contributing to mojo-websocket

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone git@github.com:YOUR_USERNAME/mojo-websocket.git`
3. Create a branch: `git checkout -b feature/your-feature`

## Development

### Prerequisites

- [Mojo](https://docs.modular.com/mojo/manual/get-started/) installed
- [pixi](https://pixi.sh/) for package management (optional)

### Running Tests

```bash
mojo run tests/test_websocket.mojo
```

### Code Style

- Run `mojo format` before committing
- Follow existing code patterns
- Add tests for new functionality
- Update documentation as needed

## Pull Request Process

1. Ensure tests pass
2. Update README.md if adding features
3. Keep PRs focused and small
4. Write clear commit messages

## Code of Conduct

Be respectful and constructive in all interactions.
