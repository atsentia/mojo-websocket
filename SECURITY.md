# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public issue
2. Email security concerns to: security@atsentia.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 48 hours and work with you to understand and address the issue.

## Security Best Practices

When using this library:

- Keep dependencies up to date
- Follow the principle of least privilege
- Validate all external inputs
- Use secure defaults provided by the library

## Acknowledgments

We appreciate responsible disclosure and will acknowledge security researchers who help improve our security.
